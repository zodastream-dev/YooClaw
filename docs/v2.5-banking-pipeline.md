# 银行模板 V2.5 — 完整情报管线

## 一、情报收集层（搜索）

```
Portal 用户 / CacheWarmer
      ↓
POST /api/portal-intel { sources: [...] }
      ↓
fetchIntelForSource(src) — 每个情报源
      ↓
V3 关键词生成：DeepSeek 根据 keywords/objects 产出 8-10 个搜索词
      ↓
合并成 3 条搜索查询 + 1 条白名单查询（site:people.com.cn OR site:caixin.com...）
      ↓
逐条查询发送到 Serper API（Google 搜索，gl=cn, hl=zh-cn, num=20）
      ↓
原始搜索结果（title, link, snippet, date）
```

**当前配置**：银行模板 4 个源全部用 `aiProvider: 'serper'`，只走 Serper 一条渠道。

---

## 二、筛选层（代码硬过滤，不消耗 Token）

```
原始搜索结果
      ↓
去重（URL/Title 相同合并）
      ↓
Old URL 噪音过滤（电商/汽车媒体/游戏等）← URL_NOISE_RULES
      ↓
传递给 DeepSeek 分析
```

**已移除的过滤**：V2.5 噪声过滤器（_noiseType 自分类）——因误杀"超大型油轮订单"等工业情报，已删除。

---

## 三、AI 分析层（DeepSeek 处理）

```
搜索上下文（最多 100 条 / 15000 字符）→ 喂给 DeepSeek
      ↓
System Prompt：src.customPrompt（银行专用提示词）
User Prompt：提取 30 条情报，JSON 格式，包含以下字段：
      ↓
DeepSeek 输出 [{title, summary, source, date, url,
  _sentiment（正面/负面/中性）,
  _reliability（已确认/传闻/待核实）,
  _intent（竞对意图）,
  _valueScore（0-100 商业价值分）,
  _riskLevel（CRITICAL/WARNING/NORMAL）,
  _noiseType（对公业务/零售噪音/营销通稿）,
  _provider（从 _searchProvider 原样复制）}]
      ↓
后处理：
  - URL 幻觉校验（不在搜索结果白名单中的 URL 丢弃）
  - Date 恢复（从原始搜索匹配回填丢失的日期）
  - _provider 恢复（同上匹配回填来源渠道）
  - 旧文过滤（>365 天丢弃）
  - EHR 实体幻觉移除（对象名不在搜索结果中的丢弃）
  - 空摘要填充（从原始搜索回填）
```

**银行专属评分规则**（在 User Prompt 中注入）：
```
目标客户专项：评级下调/违约/亏损 → 90+；战略调整/融资 → 75-89
竞争对手专项：签战略协议/银团牵头权 → 90+；新产品/调整 → 75-89
```

**四维风险判定**（在 User Prompt 中注入）：
```
高管动态：离职/被查 → CRITICAL；战略表态 → WARNING
战略布局：新业务/海外扩张 → WARNING
资金成本：评级调整/千万罚单 → CRITICAL
项目动态：银团牵头权丢失/主办行变更 → CRITICAL
```

## 四、缓存与交付层

```
分析结果
      ↓
白名单可信度标注（代码层，不耗 Token）：
  → 域名在白名单（people.com.cn/caixin.com/等）→ _credibility: "HIGH"
  → .gov.cn/.edu.cn → "HIGH"
  → 新浪/东方财富/雪球 → "MEDIUM"
  → 其他 → "LOW"
      ↓
portalIntelCache（30 分钟 TTL，daily 源 = 24h 有效）
      ↓
POST /api/portal-intel → res.json({ results })
```

**更新机制**：
- **正常访问**：从缓存读取，秒级返回
- **点"立即更新"**：清缓存 + 立即返回旧数据 + 后台异步刷新
- **CacheWarmer**：每 20 分钟检查一次，只刷新已过期（>24h）的缓存

## 五、前端展示层

```
allIntelData（从 API 或 localStorage 加载）
      ↓
左侧面板：4 个情报源卡片
  - 行业信号（蓝色，无子对象）
  - 目标客户情报（四对象：中铁/中交/中建/国网）
  - 竞争对手情报（五对象：工/建/农/中/招）
  - 自身舆情（红色，六风险维度）

点击卡片 → selectSourceFilter() → 筛选中央情报流
点击对象名 → selectObjectFilter() → 展开该对象情报
      ↓
中央情报流：按 _valueScore 降序排列
  每条卡片显示：
  - 分值徽章（>=75 橙色高亮边框）
  - 来源标签（秘塔/天聚/Serper 等）
  - 可信度标签（🟢 高可信 / 🔴 待验证）
  - 风险等级（🚨 高危红色脉冲 / ⚠ 预警橙色）
  - 情感倾向（正面/负面/中性）
  - 对象标签（中国中铁 / 工商银行 等）

高危卡片：红色左边框 + CSS 脉冲动画
      ↓
点击卡片 → 新标签页打开原文 URL
```

## 六、当前积分消耗预估

| 环节 | 积分类型 | 每轮消耗 |
|------|---------|---------|
| 搜索（Serper） | Serper 配额 | 4 源 × 5 查询 × 1 次 = ~20 次 |
| AI 分析（DeepSeek） | DeepSeek Token | 4 源分析（~2-3 元/轮） |
| 关键词生成（V3） | DeepSeek Token | ~16 次轻量调用 |

每日自动更新 1 次（daily），CacheWarmer 检查 72 次但只触发达标者。
