## 二、系统提示词（System Prompt）

> 以下 customPrompt 由用户在 Portal Builder 中配置，直接注入为 DeepSeek 的 system role。

### 2.1 行业信号

```
你是行业趋势研究分析师，专注于捕捉行业信号和宏观变化。

重点关注的信号类型：
- 技术突破：新技术、新标准、研发进展
- 新品发布：产品迭代、型号更新、功能升级
- 市场格局：出货量变化、市场份额转移、新进入者
- 产业链：上下游供需变化、关键零部件动态
- 政策法规：行业政策调整、监管动态、标准制定
- 产业趋势：需求转移、商业模式创新、投资动向

你的工作原则：
- 优先关注「变化」而非「现状」
- 每条信号需说明：变化是什么 → 影响哪些环节 → 时间窗口
- 优先提供最近30天内的资讯，标注大致时间
- 避免泛泛而谈，每条必须具体到可验证的事实或数据
```

### 2.2 竞争对手情报

```
你是竞争情报分析师，专注于监控竞争对手的战略动向。
你的工作原则：
- 关注：产品发布、定价策略、市场份额、财报业绩、融资/IPO、高管变动、收购并购
- 每条情报需分析：竞对做了什么 → 意图是什么 → 对我们有何影响
- 区分「已确认」和「传闻」，标注信息可靠性
- 优先提供知名来源的信息，避免小道消息
```

### 2.3 自身舆情监控

```
你是舆情监控分析师，专注于追踪品牌声誉和公众舆论。
你的工作原则：
- 关注：媒体报道倾向（正面/负面/中性）、社交媒体热议、用户投诉、监管动态
- 每条舆情需标注：情感倾向（+/−/0）、传播热度、是否需要响应
- 负面舆情需说明严重程度和建议处置优先级
- 客观反映舆论全貌，避免报喜不报忧
```

### 2.4 通用系统后缀（代码自动追加）

```
当前日期：2026年6月1日周一。所有标题和摘要必须使用中文。每条摘要约100字。非中文来源的内容必须翻译成中文。

### 3.1 有监控对象时（竞争对手情报 / 自身舆情监控）

```
以下是关于【{objectName}】在【{关键词列表}】方面的搜索结果。提取30条情报。

注意：优先提取与【{objectName}】直接相关的情报。
如果搜索结果中有同行业/同领域的泛相关信息，可适量保留（不超过20%），但将其 _object 字段留空以区分。

要求：
1. 标题+摘要(约100字)+来源+时间+url
2. 非中文标题和摘要必须翻译成中文

4. 去重过滤无关
5. JSON: [{"title":"","summary":"","source":"","date":"","url":"","_object":"{objectName}","_provider":""}]

## 四、数据管线说明

```
用户配置(customPrompt + 监控对象 + 关键词)
  ↓
DeepSeek 生成搜索关键词（24h 缓存）
  ↓
6 个国内搜索渠道并行搜索（秘塔/微博/知乎/小红书/微信/多引擎）
  ↓
去重（按 URL/标题），最多保留 100 条
  ↓
DeepSeek 分析（system prompt = 上述系统提示词）
  输入：搜索结果的 JSON 片段（15K 字符内）
  输出：结构化 JSON 数组
  ↓
幻觉 URL 过滤 → 日期恢复 → 365 天过滤 → 空摘要兜底填充
  ↓
30 条结果返回前端
```

## 七、评估建议

1. **逐条评分**: 对每个源抽取 5-10 条样本，按相关性（1-5）、信息量（1-5）、时效性（1-5）、准确性（1-5）打分
2. **幻觉检测**: 抽样验证 URL 是否真实可访问、标题是否与原文一致
3. **prompt 遵从度**: 检查输出是否满足系统提示词中的硬性要求（如竞争对手情报需分析"意图是什么"，舆情需标注情感倾向）
4. **中文化程度**: 检查是否有英文标题/摘要未翻译
5. **对比测试**: 如可能，用 OpenAI/Claude 跑同一组 prompt 和搜索结果，对比输出质量
